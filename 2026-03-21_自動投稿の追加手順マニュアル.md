## 方法 B：新しく「次の2週間用ファイル」を作る

ファイルが長くなりすぎるのが嫌な場合や、月ごとにファイルを分けたい場合の方法です。

### 1️⃣ 新しい投稿ファイルを作る
新たに `2026-04-06_imakentei_4月投稿案.md` などのファイルを作り、中身を `### Day1` から書き始めます。
※画像がある場合は `投稿用画像/2026-04-06.png` のように配置してください。

### 2️⃣ `gen_json.py` の設定を書き換える
`gen_json.py` ファイルを開き、**一番上にある設定部分（6〜9行目付近）** を書き換えます。

```python
# --- 設定 ---
# 読み込むMarkdownファイルの名前
TARGET_MD = '2026-04-06_imakentei_4月投稿案.md'    # ← ここを新しいファイル名にする

# Day1 の投稿開始日 (YYYY-MM-DD形式)
START_DATE = '2026-04-06'                        # ← ここを新しい Day1 の日付にする
# -----------
```

### 3️⃣ データを変換する
ターミナルで `python gen_json.py` を実行します。
これで「新しいMarkdownファイル」を元にして、日付や時間が更新された `post_data.json` が生成されます。
**※注意**: 過去の投稿履歴（`is_posted` フラグ）は自動で引き継がれます。

### 4️⃣ GitHubにアップロード（プッシュ）する
`post_data.json` や新しいMarkdownファイル、`gen_json.py` の変更を `git push`（アップロード）します。
これで新しい期間の予約投稿がセットされます！

## 🕒 よくある質問

* **投稿はいつされる？**
  * 朝 7:00 と 昼 12:00（GitHub版につき、1〜5分程度の遅れが発生する場合があります）。
* **パソコンの電源は？**
  * クラウド（GitHub Actions）で動いているので、**PCの電源が切れていても、スリープしていても必ず実行されます。**
* **ちゃんと動くか事前にテストしたい**
  * GitHub ブラウザ画面の **「Actions」** タブ → **「 IMA検定 自動投稿（日次7時・12時）」** → **「Run workflow」** から手動でテスト実行ができます（フライング投稿はされない安全設計です）。
